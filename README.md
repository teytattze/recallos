# RecallOS

> Shared memory for your organization — one place to store everything, one API for humans and AI agents to recall it.

## Overview

RecallOS is your organization's memory. It holds all of your documents, messages, processes, and other knowledge in a single place. Anyone — people and AI agents alike — can read from and write to this shared memory to retrieve the right information at any point in time.

## Axioms

The assumptions RecallOS is built around:

- AI agents always start with fresh context.
- AI agents have a limited context window.
- Every piece of information is related to other pieces of information.

## Problems

- Information is scattered across many different places.
- Related entities aren't mapped with meaningful relationships.
- There is no single API to capture all of this information.

## Solutions

RecallOS centralizes all information in one place and exposes a single API for humans and AI agents to read and write it. The result:

- Less time spent searching for information.
- More accurate, more trustworthy information.
